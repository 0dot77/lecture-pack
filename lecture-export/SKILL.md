# /lecture-export — 강의 슬라이드 PDF 익스포트

> **역할**: HTML 슬라이드를 고화질 PDF로 변환
> **트리거**: `/lecture-export` 또는 PDF 익스포트 요청
> **출력**: 고화질 PDF 파일
> **의존**: Node.js, Playwright (자동 설치)

## Purpose

`/lecture-slides`나 `/frontend-slides`로 만든 HTML 프레젠테이션을 인쇄 및 배포 가능한 고화질 PDF로 변환한다.

## Behavior

### Phase 1: 입력 확인

**Q1 — 대상 파일**
> "어떤 HTML 파일을 PDF로 내보낼까요?"
>
> 현재 디렉토리에 HTML 프레젠테이션이 있으면 자동으로 감지하여 제안한다.

**Q2 — 화질 선택**
> "화질을 선택하세요:"

| 옵션 | 해상도 | 스케일 | 용도 | 예상 크기 |
|------|--------|--------|------|----------|
| **고화질 (기본)** | 1920×1080 | 2x (Retina) | 화면 공유, 고화질 인쇄 | 슬라이드당 ~2-3MB |
| **초고화질** | 2560×1440 | 2x (Retina) | 대형 인쇄, 포스터 | 슬라이드당 ~4-5MB |
| **표준** | 1920×1080 | 1x | 이메일 첨부, 빠른 공유 | 슬라이드당 ~1MB |
| **경량** | 1280×720 | 1x | 모바일 열람, 용량 절약 | 슬라이드당 ~0.5MB |

**Q3 — 출력 경로 (선택)**
> "PDF를 어디에 저장할까요? (기본: HTML 파일과 같은 위치)"

### Phase 2: 익스포트 실행

이 스킬의 `scripts/export-pdf-hq.sh` 스크립트를 사용하여 익스포트한다.

```bash
bash <skill-path>/scripts/export-pdf-hq.sh <html-path> <output-path> --quality <quality>
```

**스크립트 동작:**
1. 로컬 HTTP 서버 시작 (폰트/에셋 로딩을 위해)
2. Playwright로 헤드리스 브라우저 실행
3. 각 슬라이드를 지정된 해상도와 스케일로 스크린샷
4. 폰트 로딩, 애니메이션 정착 대기
5. `.reveal` 요소 강제 표시 (애니메이션 상태와 무관하게)
6. 모든 스크린샷을 PDF로 조합
7. 임시 파일 정리 후 PDF 열기

### Phase 3: 결과 전달

- PDF 파일 경로와 크기를 안내한다
- 자동으로 PDF를 연다
- 참고 사항을 안내한다:
  > - 애니메이션은 정적 상태로 캡처됩니다
  > - 내장 영상은 첫 프레임 또는 썸네일로 표시됩니다
  > - 외부 이미지 로드 실패 시 fallback 텍스트가 표시됩니다

## Notes

- 기본 화질은 고화질(1920×1080, 2x)이다 — 대부분의 강의 배포에 충분
- 초고화질은 대형 프린트나 포스터 용도에만 권장
- `--compact` 플래그로 경량 모드 사용 가능
- Playwright 첫 실행 시 Chromium 다운로드에 시간이 소요될 수 있음
